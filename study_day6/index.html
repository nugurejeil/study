2. CSS 퀄리티를 높히는 숩관

  • CSS문서 최 상단에 @charset "utf-8"; 설정으로 문자가 깨지지 않도록 하기

  • reset.css가져오기 (Eric Meyer reset.css 등등)

  • CSS validator 통과하기

  • CSS 작성 순서 정하기

    1. 레이아웃에 큰 영향을 주는 요소들 : float, display, width, height, padding, margin, border

    2. 폰트 요소들 : font-size, line-height, font-family, letter-spacing, 기타 폰트 속성들

    4. animation, transistion, transform

    작성 순서와 관련된 자세한 얘기는 여기를 참고하세요. http://www.clearboth.org/ordering_properties/

3. CSS 방법론

정답이 없는 애매한 언어인 CSS이기에 별다른 생각없이 작성하다 보면 위와 같이 코드가 꼬이고 가독성이 떨어지게 되는 상황을 많이 접하게 됩니다. 때문에 일정한 작성 기준을 정해서 코드의 유지보수를 용이하게 하고자 하는 움직임이 나타났습니다.

SMACSS(Scalable and Modular Architecture for CSS) : 대표적으로 많이 쓰입니다.

https://smacss.com

  • 조나단 스눅(Jonathan Snook)이 개발

  • 확장가능한 모듈형태의 아키텍쳐

  • 클래스명을 의미나, 목적에 맞춰서 사용(딱 봤을때 어디서 무엇을 위해 사용하는 클래스인지 분명히 알 수 있도록)

  • 카테고리 분리하기

    1. base css :html, body, header나 footer, clear fix, 기타 공통으로 쓰이는 CSS들 (클래스나 아이디를 설정하지 않아도 될 것들)

    2. layout css : 레이아웃과 관련된 CSS (큰 단위의 레이아웃). 요소들을 어떻게 나열할지 결정합니다. 보통 ID셀렉터를 이용하는데 페이지에서의 중요도를 어필하기 위함입니다.

    3. module css : 반복적으로 재사용되는 것들이 이에 해당됩니다. 어디서든 사용되기 때문에 고유의 클래스 이름을 지정해줍니다.

    4. state css : 요소의 상태에 대한 CSS(가령 버튼일 눌렸을 때의 CSS, 에러가 발생했을 때의 상태, display: none;)

    5. theme css : 특정한 클래스별로 테마를 별도로 줘야할 때
